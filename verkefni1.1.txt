1. Hvað gera eftirfarandi Linux skipanir?
cd - Stendur fyrir "change directory". Færir þig um möppu. t.d. cd Verkefni1/verkefni1.1
ls - Stendur fyrir "list". Skrifar niður allar möppur og öll skjöl í möppunni sem þú ert staddur.
pwd - Stendur fyrir "print working directory". Prentar út möppuna sem þú ert staddur í.
mkdir - Stendur fyrir "make directory". Býr til nýja möppu(a.k.a directory).

2. Hvað gera eftirfarandi Git skipanir?
git clone - Downloadar verkefni og allri útgáfu sögu verkefnisinns
git status - Skrifar allar nýjar eða breyttar möppur sem á eftir að committa
git diff - Sýnir munin á möppum

3. Hvað gera eftirfarandi Git skipanir saman? Hvaða gagn er að þeirri aðgerð?
git log - Sýnir lista af öllum commits sem þú hefur gert
git checkout - Skiptir yfir í tilgreinda grein og uppfærir directory-ið
git branch - Skrifar lista af öllum greinum

4. Hvað er útgáfustýring (Version Control)?
Version control er kerfi sem t.d. GitHub notar þar sem að það vistar allar útgáfur af skjalinu sem þú uploadar og leifir
þér að skoða gamlar útgáfur af skjalinu. Þetta getur hjálpað mikið ef þú klúðrar einhverju því þá getur þú fengið gamla 
útgáfu af skjalinu og byrjað þaðan aftur í staðinn fyrir að finna villurnar í skjalinu sem gæti tekið smá tíma. 

5. Hverjir eru helstu kostir við að nota GIT?
GIT gerir það mikið léttara og öruggara að geyma gögn. Eina sem þarf til að komast í gögnin er netsamband og einhvernveginn 
tæki(t.d. síma eða fartölvu). Ef tölvan með öllum gögnunum þínum eyðileggst þá getur þú bara farið á github.com og sótt þau
aftur. Maður veit aldrei hvað getur gerst og þess vegna er betra að taka 2 mínútur í að henda gögnunum þínum á github og 
vita að gögnin eru örugg.

6. Hversu oft telur þú að eigi að gera færslur (commit) í verkefni?
Mér finnst að í hvert skipti sem þú leysir einhvern einn part af verkefninu þá ættir þú að commita. Dáldið eins og hér
þar sem ég commita eftir hverja spurningu og þé er ekki séns að ég missi neina vinnu.
